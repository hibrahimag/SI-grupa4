# Sprint cilj

Cilj četvrtog sprinta je formalizirati kriterije završetka rada kroz Definition of Done, definisati inicijalni plan isporuke (Initial Release Plan) te postaviti tehničke temelje sistema kroz kreiranje osnovnog skeletona projekta i strukture repozitorija, uz završnu validaciju human-first faze kako bi se omogućio kontrolisan prelazak u AI-enabled fazu razvoja.


# Ključne stavke koje tim želi završiti

- Ažuriranje i korigovanje arhitekture sistema
- Definisati Definition of Done za user stories i inkremente
- Kreirati Initial Release Plan sa okvirnim rasporedom isporuka
- Postaviti osnovni tehnički skeleton sistema (backend/frontend struktura)
- Kreirati repozitorij i definisati osnovnu strukturu projekta
- Uspostaviti osnovni development setup (okruženje, alati)
- Verifikovati usklađenost implementacije sa prethodno definisanom arhitekturom
- Izvršiti završnu validaciju human-first faze


# Rizici i zavisnosti

## Rizici

- Pogrešne odluke u ranoj fazi dizajna sistema teško je ispraviti 
  bez značajnog refaktorisanja u kasnijim sprintovima
- Neusklađenost između članova tima oko tehničkih odluka 
  (arhitektura, tehnologije) može usporiti dalji razvoj
- Izbor tehnologija sa kojima tim nije dovoljno upoznat 
  povećava vjerovatnoću tehničkih problema tokom implementacije

## Zavisnosti

- Prethodno definisana testna strategija, domain model i arhitektura sistema
- Usaglašen Sprint 2 sa definisanim user stories i acceptance kriterijima
- Dostupnost i angažovanost svih članova tima 
  pri donošenju dizajnerskih odluka
- Usaglašene tehničke odluke donesene u prethodnim sprintovima
