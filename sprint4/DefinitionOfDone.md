# Definition of Done
## Sistem za upravljanje studentskim praksama

- skup kriterija koji moraju biti ispunjeni kako bi se određena stavka smatrala završenom 


  ---


| Kategorija | Kriterij |
|----------|--------|
| **Funkcionalnost** | Funkcionalnost je implementirana ili dokumentovana u skladu sa definisanim zahtjevima |
| **Funkcionalnost** | Svi definisani Acceptance Criteria su u potpunosti ispunjeni |
| **Funkcionalnost** | Funkcionalnost podržava procese sistema (prijava, odobravanje, praćenje, evaluacija) |
| **Poslovna pravila** | Pravila sistema su ispoštovana (npr. prijava samo na dostupne prakse) |
| **Uloge** | Uloge korisnika pravilno implementirane (student, kompanija, administrator, koordinator) |
| **Code Review** | Implementacija je pregledana od strane tima |
| **Code Review** | Pull request je odobren i svi komentari riješeni |

---

## Testiranje i kvalitet

| Kategorija | Kriterij |
|----------|--------|
| **Testiranje** | Funkcionalnost je adekvatno testirana |
| **Testiranje** | Unit testovi su napisani i uspješno prolaze |
| **Testiranje** | Integracijski testovi pokrivaju ključne tokove sistema |
| **Testiranje** | UI testovi potvrđuju ispravno ponašanje interfejsa |
| **Testiranje** | Pokriveni su edge case-ovi (neispravni unosi, prazna polja, neautorizovan pristup) |
| **Testiranje** | Testovi su evidentirani uz rezultate |
| **Kvalitet** | Nema kritičnih i visokih bugova |
| **Kvalitet** | Poznati problemi su dokumentovani (ako postoje) |

---

## Tehnička ispravnost

| Kategorija | Kriterij |
|----------|--------|
| **Backend** | API endpointi rade prema specifikaciji |
| **Backend** | Validacija podataka na backendu je implementirana |
| **Podaci** | Podaci se ispravno čuvaju i ažuriraju u bazi |
| **Sigurnost** | Autentifikacija i autorizacija rade ispravno |
| **Sigurnost** | Osnovne sigurnosne mjere su implementirane |
| **Kod** | Kod prati standarde i komentarisan je gdje je potrebno |
| **Integracija** | Funkcionalnost je povezana sa ostatkom sistema |
| **Build** | Build prolazi bez grešaka |
| **Repozitorij** | Kod je pushan na repozitorij |
| **Performanse** | Nema očiglednih problema sa performansama |
| **Sistem** | Sistem se može pokrenuti bez problema kod svih korisnika |

---

## Frontend i UX

| Kategorija | Kriterij |
|----------|--------|
| **UI** | UI odgovara dizajnu i funkcionalnim zahtjevima |
| **Validacija** | Forma validacije radi ispravno (obavezna polja, format email-a) |
| **UX** | Greške se prikazuju korisniku na jasan način |
| **UX** | Aplikacija je upotrebljiva (osnovni UX zadovoljen) |

---

## Dokumentacija i artefakti

| Kategorija | Kriterij |
|----------|--------|
| **Artefakti** | Promjene su evidentirane u relevantnim projektnim artefaktima |
| **Backlog** | Product Backlog i Sprint Backlog su ažurirani |
| **Dokumentacija** | Dokumentacija je ažurirana gdje je potrebno |
| **Odluke** | Tehničke i dizajnerske odluke su evidentirane u Decision Log-u |
| **AI** | AI Usage Log je ažuriran  |

---

## Spremnost za isporuku

| Kategorija | Kriterij |
|----------|--------|
| **Demo** | Funkcionalnost je spremna za demonstraciju u sprint review-u |
| **Demo** | Funkcionalnost se može demonstrirati bez dodatnih izmjena |
| **Deployment** | Funkcionalnost je spremna za pokretanje u predviđenom okruženju |
| **Validacija** | Product Owner potvrđuje da je User Story završen |
